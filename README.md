# לוח בקרת קטלוג — דשבורד ניהול ממצאים

דף חי לניהול מעגל-הבקרה המלא של פרויקט הבקרות: קליטת תוצרי הריצה בגרירה,
מעקב למי נשלח, חיווי שנצפה, סטטוס טיפול, תזכורות באיחור, ותפוקת אנליסטים.

## הרצה מקומית
```
py -m pip install -r requirements.txt
py -m streamlit run app.py
```

## פריסה בענן (בסגנון smart-cataloger)
1. העלה את התיקייה לריפו GitHub (ranmf21-hub/baqarot-dashboard).
2. share.streamlit.io → New app → בחר את הריפו ואת app.py.
3. בענן אין תיקייה שיתופית — עובדים במצב העלאה/הורדה של המאגר (הכפתורים בסרגל הצד).

## המאגר (מקור האמת)
`מאגר בקרות.xlsx` — יושב בתיקייה השיתופית (מוגדר בסרגל הצד ונשמר ב-config.json).
שלושה גיליונות: ממצאים / משלוחים / תפוקה. כל שמירה יוצרת גיבוי מתוארך בתיקיית "גיבוי מאגר".

## מה גוררים פנימה (טאב "קליטה בגרירה")
| קובץ | מה קורה |
|---|---|
| `mail_queue.txt` מהריצה | ממצאים חדשים + נתוני תפוקה נקלטים (כפילויות מדולגות) |
| `בקרה - קטלוג שוטף … (vbs).xlsx` | ממצאים מחולצים ישירות מקובץ הבקרה |
| `תפוקת אנליסטים ….xlsx` | טבלת התפוקה נקלטת לטאב התפוקה |
| מייל שנשלח (`.msg`/`.eml`) | הממצאים של הנמען+התקופה מסומנים "נשלח" + נרשם במשלוחים |
| מענה של אנליסט (`RE: …`) | הממצאים שלו מסומנים "נצפה" עם תאריך ומקור |

גרירת מייל מ-Outlook: גרור את המייל לשולחן העבודה (נוצר `.msg`) ומשם ללוח.

## מחזור חיים של ממצא
פתוח → נשלח → נצפה → בטיפול → טופל / לא רלוונטי.
"באיחור" = פתוח/נשלח/נצפה מעל N ימים (ברירת מחדל 7, מוגדר בסרגל הצד) → נורה 🔴 ורשימת תזכורות בסקירה.
